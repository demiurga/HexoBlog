learning es6 with babel - const

203月

const用來聲明常量，也就是該變數的值在聲明後不會再改變，跟let一樣有block scope，沒有
hoisting

babel會在compile階段就檢查PI是不是在宣告後有試圖去改變他，所以上面這段會出現Error

由於常量的特性是在compile階段檢查的，所以babel對於const其餘特性(block scope,
no hoising)的實現方式就跟let一樣

要注意的是const不等於Object.freeze()：const是固定變數的參考，Object.freeze()
則是固定變數參考對象的值

learning es6 with babel - let

203月

let是另一種宣告變數的方式，他有幾個特點：有block scope，沒有hoisting，不能重複宣告

babel對他的實現方式是在前面加個prefix_

let不像var一樣具有hoisting的特性，這造成一些現象

在一個block scope中，只要有進行let宣告的變數，那麼該變數在被宣告前都不能被使用。
這個區塊就被稱為temporal dead zone

babel的實現方式如下

他一開始先在全域宣告一個_temporalUndefined物件作為not defined flag，當我們在block scope中做let宣告時，
他會先在scope的開始部分就先用var宣告物件，然後指向全域的_temporalUndefined物件。接下來
在每次調用x時，都會先檢查他的參考對象是不是_temporalUndefined，如果是的話就拋出
ReferenceError

可以注意的點是，val === undef比較的是參照，而不是值，所以就算你用let x={}也不會怎樣，
因為這兩個{}是不同的匿名物件

當撞名時babel會自動改名，不必擔心

“不能重複宣告”這項特性則是在babel compile階段做檢查的

Learn es6 with babel - Arrows

153月

在ES6中，可以使用=>來定義function。這是一個很類似coffeescript的語法糖（事實上es6有很大一部分是語法糖，個人很喜歡這種做法，與其去改變語言的本質，不如讓語言順著原本的脈絡慢慢茁壯）

當有參數，小括號可以省略

當return不是單一Expression時，需要用大括號把它包起來，就像一般在寫function一樣

它可以讓code更加的簡短且語意化，例如用在callback上

有種lambda的感覺，對吧？
要注意的是，如同在coffeescript中一樣，fat arrow(=>)除了是function的縮寫之外，還具有在定義時綁定this的特性
